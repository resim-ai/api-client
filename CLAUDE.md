## What this is

The ReSim CLI (the `resim` binary): a Go command-line tool for the ReSim API. Most of `api/` and `bff/` is **generated code** — the REST client is generated from the rerun OpenAPI spec via `oapi-codegen`, and the BFF GraphQL client via `genqlient`. Module: `github.com/resim-ai/api-client` (Go 1.23).

## Commands

- Build:   `go build -o resim ./cmd/resim`   (CI builds all: `go build ./...`)
- Test:    `go test ./... -race`             # single test: `go test -v -run TestName ./path/to/pkg`
- E2E:     `go test -v -tags end_to_end -count 1 ./testing`   # needs RESIM_CLIENT_ID / RESIM_CLIENT_SECRET + CONFIG (staging|prod)
- Lint:    `go fmt ./...` and `go vet ./...`  # run BEFORE pushing — CI fails if either emits any output
- Codegen: `go generate ./...`               # regen after OpenAPI / GraphQL schema changes; generated files are committed

There is no Justfile or Makefile — drive everything with the `go` toolchain directly. `staticcheck.conf` configures staticcheck (whitelists the dot-import of `ptr`) if you choose to run it; it is not part of the CI gate.

## PRs

Uses **Graphite**: `gt track` (before the first push on a new branch), then `gt submit`. Never `git push` + `gh pr create` here.

## Testing conventions

- Standard Go `testing` package; `*_test.go` files sit next to the code they cover (e.g. `cmd/resim/commands/*_test.go`).
- CI runs the full suite with `-race`.
- The end-to-end suite in `testing/` is gated behind the `end_to_end` build tag, so it is skipped by a plain `go test ./...`. It hits a live deployment and needs client credentials plus `CONFIG` (and `DEPLOYMENT` for a customer dev deployment). Fixtures live in `testing/data/`.
- API-client mocks are generated by mockery into `api/mocks/` (config: `api/.mockery.yaml`) — do not hand-write them.

## Gotchas

- **Generated code is committed and large — never hand-edit it.** `api/client.gen.go` (~1 MB), `bff/queries.gen.go`, and `api/mocks/` are all outputs of `go generate ./...`. Change the source (OpenAPI spec / `bff/queries/` GraphQL / mockery config) and regenerate instead.
- **`go generate` fetches the spec over the network.** `api/generate.go` pulls the OpenAPI spec from `https://api.resim.ai/v1/openapi.yaml` (production URL) — regen needs internet access, and the client tracks whatever the prod spec currently is.
- The `ptr` package is intentionally dot-imported; staticcheck is configured to allow it. Keep new dot-imports out.
- The built `resim` binary and `.env.local` are gitignored — don't commit them.

## Directory map

- `cmd/resim/`         — CLI entrypoint (`main`); `cmd/resim/commands/` holds the cobra commands (agents, batch, build, experience, metrics, project, sweep, test_suites, …) with adjacent `_test.go` files.
- `api/`               — generated REST client (`client.gen.go`), `generate.go` (oapi-codegen + mockery directives), `client.cfg.yml`, `.mockery.yaml`, and `mocks/`.
- `bff/`               — GraphQL BFF client: `queries/` (source `.graphql`), `queries.gen.go` (genqlient output), `genqlient.yaml`, `cmd/` (the generator).
- `auth/`              — authentication (token caching, client-credentials / username-password flows).
- `ptr/`               — pointer helper utilities (dot-imported).
- `testing/`           — end-to-end test (`end_to_end` tag) and `data/` fixtures.
- `docs/plans/`        — per-repo plan documents.
- `.github/workflows/` — CI: lint-and-build, build-image, release-cli.
- `.devcontainer/`     — dev container config.
